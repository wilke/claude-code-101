# 2026-05-12 — The leakage dead end (twice)

Spent today chasing a "great" model that turned out to be two different leaks. Both
are worth recording so nobody re-walks them.

## Leak 1 — the `outcome_label` column

Early run reported **accuracy ≈ 1.00, ROC-AUC ≈ 1.00** across every fold. That's not
a good model, that's a tell. Cause: `outcome_label` was left in the feature matrix.
It's just `outcome` spelled `success`/`failure` — after one-hot encoding, the model
"predicts" the target from a renamed copy of itself.

- **Dead end.** Any feature set that includes `outcome_label` (or anything derived
  from `outcome`). Fix: drop it before building `X`, and keep an
  `assert "outcome_label" not in X.columns` guard so the mistake fails loudly.
- **How it was caught.** A perfect CV score. Rule going forward: **treat ≈1.00 as a
  leak until proven otherwise**, never as success.

## Leak 2 — preprocessing fit before the split

Subtler. After removing `outcome_label`, CV F1 looked a little *too* good (≈ 0.72)
but the held-out **test F1 collapsed to ≈ 0.55**. Cause: `StandardScaler` and
`SimpleImputer` were fit on the **whole** dataset *before* the train/test split (and
an experiment with SMOTE resampling on the full data made it worse). The scaler's
mean/variance and the imputer's medians had already "seen" the test rows, so CV was
optimistic.

- **Dead end.** Fitting any stateful step (scale, impute, resample) on the full data
  before splitting. It inflates CV and the gap only shows up on a truly held-out set.
- **Fix.** Move **every** fitted step inside a single sklearn `Pipeline` /
  `ColumnTransformer`, so cross-validation re-fits it on each fold's training portion
  only. Row-level, fit-free cleaning (dedupe, label-normalize, parse concentration,
  quarantine the 9999) may stay before the split — it learns no parameters.
- **After the fix.** CV F1 ≈ 0.61 and held-out test F1 ≈ 0.60 — lower, but *honest*,
  and now they agree.

**The lesson in one line:** a model result that looks great and a model result you
can trust are different claims; the split-before-fit discipline is what turns the
first into the second.
