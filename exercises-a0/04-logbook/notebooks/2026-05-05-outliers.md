# 2026-05-05 — The 9999 temperature and the missing cells

Chased the temperature column today. Two problems: a handful of blank
`temperature_C` cells, and one row reading **9999**. The 9999 alone pushed the raw
mean from ~37 °C to ~60 °C, so it has to be dealt with before any summary.

- **Decision (temperature outlier).** A lab temperature of 9999 °C is physically
  impossible, and we can't recover the true value from the sheet, so quarantine
  it: set any `temperature_C > 200` to `NaN` and report the count. Do **not**
  guess a replacement. Threshold 200 °C is comfortably above any real reading here
  (all others are ~30–45 °C).
- **Decision (missing cells).** Keep blank `temperature_C` and `concentration` as
  `NaN` — do not drop the rows. `yield`/`outcome`/`treatment_group` are still
  usable for those rows, and pandas skips `NaN` in means anyway. Report the
  missing count per column so nothing is hidden.
- **Parameter.** Outlier threshold `temperature_C > 200 °C → NaN`.

**Open question:** is `9999` a *sentinel for missing* (some instruments write 9999
for "no reading") or a fat-fingered **99.9**? If it's a sentinel we should scan for
other sentinel values (−1, 999) across all numeric columns; if it's a typo we
might recover 99.9. Ask whoever exported the data before deciding to repair vs
drop.
