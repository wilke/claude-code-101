# Plan: Implement primitives for the ADMM-Filter algorithm

## Context

`RASC-MeetingNotes.tex` describes a filter-based variant of the existing ADMM driver in `admm.py`. The filter algorithm adaptively steers the penalty parameter `ρ` using two stationarity measures — primal infeasibility `η` and dual infeasibility `ω` — and a non-dominated filter of past `(η, ω)` pairs to reject bad steps.

None of this currently exists. The existing code (`admm.py`, `reciprocal_approx.py`, `graph_tv.py`, `compliance.py`) implements the inner subproblem solvers but has no `η`/`ω` evaluation, no filter, and no projection onto `X = {x ∈ [ε,1]ⁿ : 1ᵀx ≤ B}`.

This plan covers the **first pass**: the three reusable primitives that the outer ADMM-Filter driver will need. The outer driver itself (Algorithm in §2.5 of the notes) is deferred to a follow-up so each primitive can be validated in isolation first.

**Decisions already made with user:**
- `ω` for the non-smooth `α·TV(y)` term uses a proximal-gradient residual, not a subgradient.
- Scope: Filter + projection + `η`/`ω` evaluator. No new outer driver yet.

### 1. `filter.py` — `Filter` class

Implements §2.1 ("Filter Entries") of `RASC-MeetingNotes.tex`.

```
class Filter:
    def __init__(self, beta=0.99, gamma=1e-5)
    def add(self, eta, omega)            # only if eta > 0; prune dominated
    def is_acceptable(self, eta, omega) -> bool
    def eta_min(self) -> float           # smallest η in the filter (used by restoration switch later)
    def __len__(self), __iter__(self)    # for inspection / debugging
```

Acceptance rule (Eq. (filter) in the notes): `(η, ω)` is acceptable iff for **every** stored entry `(η_l, ω_l)`:
```
η ≤ β·η_l   OR   ω ≤ ω_l − γ·η
```
Empty filter ⇒ everything acceptable.

Dominance pruning on `add`: after appending `(η, ω)`, drop any stored `(η_l, ω_l)` with `η_l ≥ η` AND `ω_l ≥ ω` (the new entry dominates them). Skip the add entirely if `η == 0` (per the §2.5 footnote: "ensures η_l > 0").

Self-test in `__main__`: build a filter with several entries, verify pruning, verify acceptance for boundary cases (empty filter, single entry, dominated point, point on boundary of acceptance region).

### 2. `projection.py` — Box + budget projection

Solves
```
min_p  ½‖p − s‖²   s.t.   p ∈ [x_lo, x_hi]ⁿ,   1ᵀp ≤ B
```
Closed-form per coordinate: `p_v*(λ) = clip(s_v − λ, x_lo, x_hi)`. The aggregate `Σ p_v*(λ)` is monotone-decreasing in `λ ≥ 0`, so:
- If `Σ clip(s, x_lo, x_hi) ≤ B`, return `clip(s, x_lo, x_hi)` (constraint inactive, `λ=0`).
- Else find `λ* > 0` by bisection (`scipy.optimize.brentq`) on `Σ clip(s − λ, x_lo, x_hi) − B = 0`.

```
def project_box_budget(s, budget, x_lo=0.0, x_hi=1.0, lam_tol=1e-12) -> (p, lam)
```

**Reuse pattern from `reciprocal_approx.solve_subproblem`** (`reciprocal_approx.py:70`): same exponential bracket search + `brentq` skeleton, but the per-coordinate map is just `clip(s − λ, x_lo, x_hi)` instead of the cubic Newton solve. Don't call `solve_subproblem` directly — that would force a degenerate cubic with `c=0` and the unnecessary `_solve_cubics` Newton loop. A dedicated 30-line function is cleaner and faster.

Self-test in `__main__`:
- Inactive case: small budget-loose problem, verify result equals `clip(s, x_lo, x_hi)`.
- Active case: verify `1ᵀp = B` to tolerance and that `(s − p)_v` has the KKT sign pattern (constant where `p ∈ (x_lo, x_hi)`, larger at the lower-clipped coords, smaller at upper-clipped).
- Cross-check against `scipy.optimize.minimize(method='SLSQP')` on a tiny `n=10` problem.

### 3. `optimality.py` — `η` and `ω` evaluators

```
def compute_eta(x, y) -> float
def compute_omega(x, y, lam, fem, edges, alpha, rho, budget, x_lo,
                  tau=1.0, cp_tol=1e-8, cp_max_iter=2000) -> float
def compute_lagrangian(x, y, lam, F_val, tv_val, rho) -> float
```

`compute_eta`: `‖x − y‖₂²`, exact per Eq. (η).

`compute_lagrangian`: helper for upcoming sufficient-decrease checks:
`L_ρ = F + α·TV − λᵀ(x − y) + (ρ/2)‖x − y‖²`. Pure arithmetic on already-computed `F_val` and `tv_val` so callers can avoid double FEA solves.

`compute_omega` (this is the new piece — extends Eq. (ω) to handle non-smooth TV):

- Smooth gradients of `L_ρ`:
  - `∇_x L_ρ = ∇F(x) − λ + ρ(x − y)`  (from `fem(x)` → returns `(F, ∇F)`)
  - `∇_y L_smooth = λ + ρ(y − x)`  (TV piece is non-smooth — handled by prox below)
- x-residual (smooth, projected gradient): `r_x = project_box_budget(x − τ·∇_x L_ρ, budget, x_lo, 1.0) − x`
- y-residual (proximal gradient): `r_y = prox_{τ·αTV + ι_Y}(y − τ·∇_y L_smooth) − y`
  where the prox is exactly the existing `chambolle_pock_graph_tv` y-subproblem with
  - `a = 0.5 · ones(n)`,  `b = −(y − τ·∇_y L_smooth)`,  `alpha → τ·α`,  `budget → B`,  `x_lo → ε`.
- Return `‖r_x‖² + ‖r_y‖²`.

`τ = 1` matches Eq. (ω) literally (no step-size in the formula). Expose it as a parameter so it can be tuned later if needed.

Self-test in `__main__`: run the existing `admm_compliance_tv(nelx=30, nely=10, …)` from `admm.py:209` for ~30 iterations, evaluating `η` and `ω` at each iterate. Both should monotonically decrease (modulo small jitter) toward zero. Also: at the first iterate (`x = y = uniform`, `λ = 0`), `η = 0` exactly, and `ω` should be the projected-gradient residual of the SIMP problem at the uniform design.
