# Plan: Movie Generation for FilterADMM Iterates

## Context

`FilterADMM.py` already saves per-outer-iteration heatmap PDFs (`images/iter_NNNN.pdf`) and a
convergence plot (`images/convergence.pdf`). The RASC-MeetingNotes.tex appendix lists "Create movie
of iterations" as an open to-do item. This plan adds a `movie` option that assembles those frames
into a single animated file, using `matplotlib.animation.FuncAnimation`. The feature mirrors the
existing `plot`/`plot_inner` toggle pattern: off by default, one flag to turn it on.

## File modified

`FilterADMM.py` only. No changes to `.tex` files, notebooks, or other modules.

---

## Implementation steps

### 1. Add `_save_movie()` helper (after `_save_convergence_plot`, line 164)

```python
def _save_movie(frames, nelx, nely, image_dir, plt, mcolors, fps=2):
```

1×3 layout (x, y, λ) matching `_save_iter_heatmap`. Lambda uses a **fixed symmetric**
`vmin`/`vmax` derived from all frames so the colormap is stable across the animation
(unlike the per-frame `CenteredNorm` in the static PDFs). Tries MP4 via `FFMpegWriter`
first; falls back to GIF via `PillowWriter` if ffmpeg is absent. Returns saved path or
`None`.

Added in the `# diagnostics` block after `plot_inner=False`.

### 3. Lazy import condition extended

`if plot or plot_inner:` → `if plot or plot_inner or movie:`
(`matplotlib.animation` is imported inside `_save_movie` only when called.)

### 4. Frame collection inside the outer loop


Placed alongside the outer `_save_iter_heatmap` call (same post-multiplier-update

### 5. `_save_movie` called after the main loop

