# Plan: `FilterADMM.py` — Double-Loop ADMM-Filter Driver

_Executed 2026-05-11. Result: `FilterADMM.py` ships and converges on the
30×10 default in 26 outer iterations where baseline `admm.py` does not.
See "Post-implementation notes" at the bottom for divergences from this
plan as written._

## Context

`admm.py` runs a fixed-`ρ` ADMM on the compliance + graph-TV problem. On the
30×10 default it visibly oscillates near convergence (per STATUS.md: at
iter 27→28, `η` jumps 0.012 → 0.85 and `ω` jumps 0.005 → 0.12). The remedy
specified in `RASC-MeetingNotes.tex` §3 is a **double-loop ADMM-Filter** that
gates iterates through a non-dominated `(η, ω)` filter, runs a restoration
phase that doubles `ρ` when the iterate is too infeasible or stuck, and
terminates on `(η, ω) ≤ ε`.

The three primitives this driver needs already exist and are self-tested in
isolation (`filter.py`, `projection.py`, `optimality.py` — see §4 of
`RASC-MeetingNotes.tex`). What's missing is the outer driver that wires them
together. This plan is for that driver, in a new file `FilterADMM.py`.

## Files

**New:** `FilterADMM.py` (sibling of `admm.py`).

**Reused (no edits):**
| File | What we use |
|---|---|
| `admm.py` | `build_grid_edges(nelx, nely)` |
| `compliance.py` | `ComplianceProblem(nelx, nely, penal)` and its `__call__` |
| `reciprocal_approx.py` | `reciprocal_approximation(F_and_grad, a, b, budget, …)` for the x-update |
| `graph_tv.py` | `chambolle_pock_graph_tv(n, edges, a, b, budget, alpha, …)` for the y-update |
| `filter.py` | `Filter(beta, gamma)` — `is_acceptable`, `add`, `eta_min`, `__len__` |
| `optimality.py` | `compute_eta`, `compute_lagrangian`, `compute_omega(…, tau=1.0)` |

The x/y subproblem call signatures (uniform `a = ρ/2`, linear `b = -ρ·target`)
are identical to the parameterisation already used in `admm.py`'s loop.

## Algorithm wiring (§3.5 of `RASC-MeetingNotes.tex`)

```
fem    = ComplianceProblem(nelx, nely, penal)
edges  = build_grid_edges(nelx, nely)
filter = Filter(beta=beta, gamma=gamma)

# initial state
x, y, lam = x0, y0, lam0           # default: x0=y0=budget/n, lam0=0
rho       = rho_init
eta_k     = compute_eta(x, y)
omega_k   = compute_omega(x, y, lam, fem, edges, alpha, rho, budget, x_lo, tau=omega_tau)
filter.add(eta_k, omega_k)         # only if eta_k > 0; else skip add

for k in range(max_outer):                                  # OUTER loop
    if eta_k <= eps_opt and omega_k <= eps_opt:
        converged = True; break

    x_j, y_j         = x.copy(), y.copy()
    eta_j, omega_j   = eta_k, omega_k

    for j in range(max_inner):                              # INNER loop
        if filter.is_acceptable(eta_j, omega_j):
            break

        # --- ADMM step (same parameterisation as admm.py) ---
        y_hat = y_j + lam / rho
        a_v   = np.full(n, rho / 2.0)
        b_x   = -rho * y_hat
        L_prev = compute_lagrangian(x_j, y_j, lam, F_val_j, tv_val_j, alpha, rho)
        omega_prev = omega_j

        x_jp1, _ = reciprocal_approximation(fem, a_v, b_x, budget,
                                            x_init=x_j, max_iter=max_iter_x,
                                            x_lo=x_lo, callback=x_inner_callback)

        x_hat = x_jp1 - lam / rho
        b_y   = -rho * x_hat
        y_jp1, _ = chambolle_pock_graph_tv(n, edges, a_v, b_y,
                                           budget=budget, alpha=alpha, x_lo=x_lo,
                                           max_iter=max_iter_y, x_init=y_j,
                                           callback=y_inner_callback)

        # --- evaluate ---
        eta_jp1   = compute_eta(x_jp1, y_jp1)
        omega_jp1 = compute_omega(x_jp1, y_jp1, lam, fem, edges,
                                  alpha, rho, budget, x_lo, tau=omega_tau)
        F_jp1, _  = fem(x_jp1)
        tv_jp1    = sum_edge_abs(x_jp1, edges)
        L_new     = compute_lagrangian(x_jp1, y_jp1, lam, F_jp1, tv_jp1, alpha, rho)
        suff_dec_ok = (L_prev - L_new) >= sigma * omega_prev   # eq. (suffRed)

        # --- restoration triggers: eq. (8), eq. (9), or MaxInner reached ---
        eta_min = filter.eta_min()                             # +inf if filter empty
        trig_8  = eta_jp1 >= beta * U
        trig_9  = (omega_jp1 <= eps_opt) and (eta_jp1 >= beta * eta_min)
        trig_M  = (j + 1 == max_inner)                         # third trigger (user spec)

        if trig_8 or trig_9 or trig_M:
            # Restoration: backtrack y_hat = x_jp1 + 2^{-m}(y_jp1 - x_jp1)
            for m in range(1, max_restoration + 1):
                y_hat_m = x_jp1 + (0.5 ** m) * (y_jp1 - x_jp1)
                eta_hat   = compute_eta(x_jp1, y_hat_m)
                omega_hat = compute_omega(x_jp1, y_hat_m, lam, fem, edges,
                                          alpha, rho, budget, x_lo, tau=omega_tau)
                if filter.is_acceptable(eta_hat, omega_hat):
                    y_jp1, eta_jp1, omega_jp1 = y_hat_m, eta_hat, omega_hat
                    break
            else:
                # restoration cap hit — return diagnostic
                converged = False
                reason    = "restoration_failed"
                # ... break out of both loops ...

            rho = 2.0 * rho                                    # double ρ

        # log suff_dec failures (do NOT retry — per user spec)
        x_j, y_j, eta_j, omega_j = x_jp1, y_jp1, eta_jp1, omega_jp1

    # --- accepted iterate: multiplier update + filter add ---
    x, y       = x_j, y_j
    lam        = lam - rho * (x - y)                           # ≡ admm.py's `lam += rho*(y-x)`
    eta_k      = compute_eta(x, y)
    omega_k    = compute_omega(x, y, lam, fem, edges, alpha, rho, budget, x_lo, tau=omega_tau)
    if eta_k > 0:                                              # eq. ensures η_l > 0 in F
        filter.add(eta_k, omega_k)

    if outer_callback is not None:
        outer_callback(x.copy(), y.copy(), k)
```

**Key correspondences with the .tex pseudocode (§3.5):**
- `λ^(k+1) = λ^(k) − ρ^(k)(x^(k+1) − y^(k+1))` ↔ same sign convention as
  `admm.py` (which writes `lam += rho*(y - x)`).
- The post-multiplier-update `ω_0` claimed equal to
  `ω_{ρ_k}(x^(k+1), y^(k+1), λ^(k))` (notes after eq. (suffRed)) — we just
  recompute `ω_k` after the multiplier update; the equality is a remark, not
  something we exploit for now.
- Filter `add` only when `η > 0` is enforced both by us and by `Filter.add`
  itself (raises on `η ≤ 0`), so the explicit `if` is belt-and-suspenders.

## API surface

```python
def filter_admm_compliance_tv(
    nelx, nely, penal, alpha, budget,
    # filter-specific
    rho_init=1.0, beta=0.99, gamma=1e-5, sigma=1e-5, U=None,
    max_outer=100, max_inner=20, max_restoration=30, eps_opt=1e-3,
    omega_tau=1.0,
    # inherited from admm.py
    x_lo=1e-4, max_iter_x=100, max_iter_y=2000,
    verbose=True,
    outer_callback=None, x_inner_callback=None, y_inner_callback=None,
) -> (x, info):
    ...
```

`U` defaults to `eta_0 / beta` if not provided (so the initial iterate is on
the boundary of the restoration trigger and any worse `η` triggers it).

`info` dict (extends `admm.py`'s):
- existing: `'y'`, `'lam'`, `'n_iter'` (= outer iterations), `'converged'`,
  `'objective'` (per outer iter)
- new:
  - `'rho_history'` — `ρ_k` per outer iter
  - `'eta_history'`, `'omega_history'` — per outer iter (the accepted ones)
  - `'filter_size_history'` — `len(filter)` per outer iter
  - `'restoration_count'` — total restoration phases triggered
  - `'suff_dec_failures'` — count of `(L_prev − L_new) < σ·ω_prev`
  - `'reason'` — one of `"converged"`, `"max_outer"`, `"restoration_failed"`

## Design decisions (resolved with user)

| # | Decision | Source |
|---|---|---|
| 1 | **Sufficient-decrease failure**: log only, do NOT retry. Treat `j == MaxInner` as a third restoration trigger alongside eqs. (8), (9). | user spec |
| 2 | **MaxInner = 20**, **MaxRestoration = 30**. On MaxRestoration hit: exit with `converged=False` and `reason="restoration_failed"`; do not raise. | user spec |
| 3 | **`__main__` self-test**: 30×10 default problem, print `(k, η, ω, ρ, |F|, restoration?)` trajectory each outer iteration. Should converge without the iter-27 oscillation seen in `admm.py`. | user spec |
| 4 | **`τ` in `compute_omega`**: exposed as `omega_tau` parameter, default `1.0` (matches `optimality.py` and the literal eq. (ω)). STATUS.md flags this for re-tuning later. | user spec |

## Things worth flagging during implementation

- **Cost of `ω`**: each `compute_omega` call solves a Chambolle-Pock prox
  internally. We call it (a) at outer init, (b) once per inner step,
  (c) once per restoration backtrack. On a 30×10 problem this is cheap; on
  larger meshes it may dominate. Worth measuring before optimising.
- **Warm-starts**: `reciprocal_approximation` and `chambolle_pock_graph_tv`
  both accept `x_init`. We pass `x_j` / `y_j` from the previous inner step
  as warm-starts (matches `admm.py`'s pattern of carrying state between outer
  iterations).
- **Initial filter**: per pseudocode, `F_0 = {(η^(0), ω^(0))}` if `η^(0) > 0`,
  else empty. Empty-filter acceptance returns True (per `filter.py`), so an
  empty initial filter just means the very first ADMM step is accepted
  unconditionally — fine.
- **Convergence on `ω_k`**: outer convergence checks `omega_k` computed at
  the post-multiplier-update state; this is `ω_0` in the meeting notes'
  language (the "traditional first-order error").

## Verification

1. **`__main__` self-test** — run on the 30×10 / 40% volume default with
   `rho_init=1.0`. Print per-iteration `(k, η, ω, ρ, F, n_inner, restored?)`.
   Acceptance criterion: no `η`-jump of >10× between consecutive accepted
   iterates (the failure mode `admm.py` exhibits at iter 27).

2. **Compare to baseline** — running side-by-side with `admm.py` on the same
   problem, the FilterADMM final objective should be within 1% of the
   baseline final objective, and `η`, `ω` at termination should both be
   `≤ eps_opt`.

3. **Filter behaviour sanity** — at exit, print the filter contents. Expect
   a small Pareto-style front of `(η, ω)` pairs. Should never contain a pair
   with `η == 0`.

4. **Reproduction commands** (per STATUS.md, base python lacks scipy):

## Post-implementation notes (added 2026-05-11 after the run)

Three things diverged from the plan as written above; the file as shipped
in `FilterADMM.py` reflects the divergences below.

1. **`U` default changed to `10.0`** (the plan said `eta_0 / beta`). Reason:
   `eta_0 = 0` for uniform init, so `U = eta_0 / beta = 0` would have made
   `trig_8 = (eta_jp1 >= 0)` fire on every step.

2. **Inner-loop acceptance test moved from top of the `j`-loop to the
   bottom**. The pseudocode (and the wiring sketch above) tests
   `filter.is_acceptable` at the top, which short-circuits the loop on an
   empty filter. With `eta_0 = 0` the filter starts empty and the algorithm
   would never make progress. Now the inner loop always takes ≥ 1 ADMM
   step per outer iteration and tests acceptance afterward.

3. **Verification criterion #1 softened**. The "no η-jump of >10× between
   consecutive accepted iterates" rule is too strict — the filter
   legitimately accepts grown-η iterates when the ω-branch fires. The
   self-test now just prints the max η-jump as a diagnostic; the real
   acceptance criterion is `info['reason'] == 'converged'`.

These three deviations are recorded in LOGBOOK.md (2026-05-11 entry) and
STATUS.md ("Algorithmic departures from the §3.5 pseudocode" section).
