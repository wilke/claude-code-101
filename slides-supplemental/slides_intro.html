Prerequisites

Prerequisites at Argonne

Laptop running linux or MacOS, or access to our compute nodes

  1. ssh keys to access ANL accounts: https://help.cels.anl.gov/?s=ssh
  2. Create file ~/.ssh/config replacing YourName with your userID: 
    Host *.cels.anl.gov !logins.cels.anl.gov
    ProxyJump YourName@logins.cels.anl.gov
Host login-gce
    User YourName
    Hostname logins.cels.anl.gov
    ControlMaster auto
    ControlPersist yes
    LogLevel FATAL
Host homes-gce
    User YourName
    Hostname homes.cels.anl.gov
    ProxyCommand ssh login-gce -q -W %h:%p
    ForwardX11 yes
  3. DUO MFA set up for using ssh keys: https://help.cels.anl.gov/?s=duo+mfa

Setup

Getting Started with ClaudeCode

  1. Go to https://claude.com/product/claude-code
  2. Install: curl -fsSL https://claude.ai/install.sh | bash
  3. Copy/paste the line generated by curl to update your path
  4. Edit nano ~/.claude/settings.json by adding: 
    {
  "env": {
    "ANTHROPIC_API_KEY": "ANL-username",
    "ANTHROPIC_BASE_URL": "https://apps-dev.inside.anl.gov/argoapi",
    "CLAUDE_CODE_SKIP_ANTHROPIC_AUTH": "1"
  }
}
You're ready to use ClaudeCode!   Questions? Slack: #ai-help-no-bad-questions

Responsible use

Citing AI Use in Research

The bottom line:
Authors who use AI tools must be transparent about their usage.
Authors are fully responsible for content generated by AI tools.

SIAM

  • VP for Publications blog — A Contract of Trust
  • SIAM AI policy

AMS

  • AMS white paper on AI
  • Committee on Publications AI white paper
  • Official policy — Use of AI

Publishing elsewhere? Check your venue's policy — e.g. ACM and IEEE both have their own AI-use rules.

Note: code (and other content) generated solely by AI cannot currently be copyrighted.

This workshop walks through tools and practices — plans/, STATUS.md, LOGBOOK.md — that make AI use transparent and accountable: a durable record of what was done, when, and why.

Session management

Using Files to Track Changes

Three files to track each session

  • LOGBOOK.md — log of changes
  • plans/ — plans created by /plan
  • STATUS.md — current project status

Updated at end of each session
⇒ requirement added to CLAUDE.md

At the start of each session, read:

  1. CLAUDE.md, README.md
  2. LOGBOOK.md, STATUS.md
  3. Any LaTeX files describing the project

Optionally update CLAUDE.md:

/init

CLAUDE.md Template

Useful Addition to CLAUDE.md

Add the following to the start of your CLAUDE.md file:

## REQUIRED: Session Protocol

Mandatory rules must be followed in every session without exception.

### Session start -- read these files before doing anything else

1. `CLAUDE.md` (this file)
2. `LOGBOOK.md` -- log of prior session changes
3. `STATUS.md` -- current project status

### Session end -- before exiting, always do all three steps

1. **Update `LOGBOOK.md`** -- append a dated entry summarising this session.
2. **Replace `STATUS.md`** -- detailed capture of current state of project.
3. **Ask the user** whether a new plan file should be written to `plans/`.
